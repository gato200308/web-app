Guru App — Venta de artesanías

Aplicación multiplataforma (Flutter) para que artesanos publiquen y vendan sus productos, llegando a más personas sin importar la distancia.

Descargar APK Código (Flutter) API (backend)

Explicación

Guru App es una app de ventas de artesanías desarrollada en Flutter y con un backend en PHP desplegado en la nube. Permite a artesanos registrarse, crear y publicar sus productos (imágenes, descripción, precio), gestionar inventario, recibir pedidos y comunicarse con clientes. La idea es ayudar a los artesanos a llegar a más compradores sin importar la distancia geográfica.

Funcionalidades clave

  • Registro e inicio de sesión de usuarios (artesanos y compradores).
  • Edición de perfil y foto de usuario.
  • Publicación de productos: subir imágenes, título, descripción y precio.
  • Listado y detalle de productos con búsqueda y filtrado.
  • Carrito de compras, gestión de pedidos y historial.
  • Sistema de favoritos y reseñas de productos.
  • Integración con el backend PHP que gestiona datos y persistencia (MySQL).
  • Soporte multiplataforma: Android, Web, Windows, Linux y macOS.

Cómo conectar con tu backend

La URL base del API está configurada en el código cliente en lib/config/api_config.dart (repositorio del cliente: guru-flutter). En producción tu backend público es: https://gurubackend.usbtopia.usbbog.edu.co/. Para pruebas locales ajusta la URL al host/puerto donde ejecutes tu API (por ejemplo http://192.168.X.Y:8000 o usa 10.0.2.2 si pruebas desde el emulador Android estándar).

Instalación

  1. Descarga el APK con el botón "Descargar APK" (archivo incluido: assets/app-release.apk).
  2. En tu dispositivo Android, habilita la instalación desde orígenes desconocidos (si aplica).
  3. Abre el archivo descargado y sigue las instrucciones.

Probar con el backend (local y producción)

- En producción tu backend está en: https://gurubackend.usbtopia.usbbog.edu.co/. - Para pruebas locales ejecuta tu API (por ejemplo con PHP y Apache) en tu máquina y usa la IP de tu equipo seguida del puerto (por ejemplo http://192.168.1.5:8000) como base URL en la app.

Si pruebas en un emulador Android que corre en la misma máquina que el backend, recuerda que en emulador Android los hosts locales usan 10.0.2.2 (Android Emulator) o la IP de la máquina si usas un dispositivo físico.
